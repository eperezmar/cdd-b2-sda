2. Elementos curriculares

Competencias clave

  • Competencia digital
    • Uso eficaz de herramientas y tecnologías digitales para programar, documentar, colaborar (GitHub, Trello), probar e implementar aplicaciones web.
    • Aplicación de buenas prácticas en la seguridad y protección de datos.
  • Competencia personal, social y de aprender a aprender
    • Desarrollo de la autonomía y la responsabilidad en la gestión del propio aprendizaje.
    • Trabajo cooperativo y colaborativo para la resolución de problemas reales.
  • Competencia ciudadana
    • Compromiso con la sostenibilidad y el bienestar social mediante el desarrollo de una solución tecnológica para una entidad solidaria (banco de alimentos).
  • Competencia emprendedora
    • Iniciativa para diseñar, organizar y desarrollar un proyecto tecnológico completo.
    • Toma de decisiones y planificación de tareas en equipo.
  • Competencia en matemáticas y en ciencia, tecnología e ingeniería (STEM)
    • Aplicación de razonamiento lógico para estructurar soluciones algorítmicas y arquitecturas de software.
    • Relación práctica entre la programación y sus aplicaciones reales en el entorno social.

Competencias específicas

  • C1: Desarrollar aplicaciones web en entorno servidor con acceso a datos
  • C2: Diseñar y programar servicios web funcionales y seguros
  • C3: Trabajar en equipo en proyectos reales
  • C4: Aplicar buenas prácticas de desarrollo y documentación

Saberes básicos

  1. Acceso y gestión de bases de datos
    • Conexión de aplicaciones servidor con sistemas gestores de bases de datos relacionales (MySQL).
    • Lenguaje SQL para consultas, inserciones, actualizaciones y borrado de datos.
    • Uso de ORM (Object-Relational Mapping) como JPA/Hibernate para la persistencia de datos.
  2. Estructura y arquitectura de aplicaciones web
    • Principios del patrón de arquitectura MVC.
    • Organización de código en capas: controladores, servicios, modelos y DAOs.
    • Uso de frameworks para desarrollo estructurado (Spring Boot en el backend, AngularJS o similares en el frontend).
  3. Programación y lógica del servidor
    • Desarrollo de controladores y endpoints REST.
    • Lógica de negocio y servicios.
    • Gestión de datos persistentes mediante clases DAO y repositorios.
  4. Gestión de formularios y sesiones
    • Procesamiento de formularios web: envío, recepción, validación de datos.
    • Control de sesiones y mantenimiento del estado del usuario.
  5. Seguridad en aplicaciones web
    • Autenticación y autorización de usuarios.
    • Control de acceso a recursos según roles o permisos.
    • Protección frente a vulnerabilidades comunes (inyección SQL, CSRF, etc.).
  6. Despliegue y entrega de aplicaciones
    • Configuración de entornos locales para pruebas (Apache Tomcat, XAMPP, etc.).
    • Despliegue en servidores remotos o plataformas cloud (Heroku, Vercel, etc.).
    • Uso de herramientas de control de versiones (Git) y colaboración (GitHub, Trello).

Criterios de evaluación relacionados con las competencias específicas

C1: Desarrollar aplicaciones web en entorno servidor con acceso a datos

  • C1.1: Conecta correctamente la aplicación web con una base de datos relacional utilizando tecnologías adecuadas (por ejemplo, JDBC, JPA, Hibernate).
  • C1.2: Implementa operaciones CRUD funcionales (crear, leer, actualizar y eliminar) en los distintos módulos de la aplicación.
  • C1.3: Estructura el proyecto utilizando el patrón de arquitectura MVC separando responsabilidades de forma clara.
  • C1.4: Despliega la aplicación localmente o en un servidor remoto demostrando su funcionalidad completa.

C2: Diseñar y programar servicios web funcionales y seguros

  • C2.1: Desarrolla controladores y servicios REST que permiten el consumo de datos desde el cliente.
  • C2.2: Integra validaciones en formularios y controla correctamente errores del lado servidor.
  • C2.3: Aplica mecanismos de autenticación y autorización para proteger rutas y operaciones sensibles.
  • C2.4: Identifica y previene riesgos comunes de seguridad web (como inyecciones SQL, acceso indebido, etc.).

C3: Trabajar en equipo en proyectos reales

  • C3.1: Utiliza plataformas colaborativas (GitHub, Trello, etc.) para la planificación, seguimiento y desarrollo del proyecto en equipo.
  • C3.2: Se comunica de forma efectiva con los miembros del grupo durante el desarrollo, respetando roles y tiempos establecidos.
  • C3.3: Integra y adapta su trabajo al código del equipo asegurando consistencia y funcionalidad global.
  • C3.4: Participa activamente en la toma de decisiones técnicas del grupo justificando sus aportaciones.

C4: Aplicar buenas prácticas de desarrollo y documentación

  • C4.1: Estructura el código siguiendo convenciones y normas de estilo recomendadas para el lenguaje y framework utilizados.
  • C4.2: Documenta las funciones y módulos de su código para facilitar su mantenimiento y comprensión por parte de terceros.
  • C4.3: Utiliza control de versiones de forma efectiva: realiza commits descriptivos, gestiona ramas y resuelve conflictos.
  • C4.4: Evalúa y refactoriza su propio código aplicando criterios de eficiencia, legibilidad y escalabilidad.
