Título. Descripción de la Sda

Desarrollo de una aplicación web para la gestión del banco de alimentos

Etapa educativa y nivel: 2º DAW (Desarrollo de Aplicaciones Web)
Módulos implicados: 

  • Bases de datos
  • Despliegue de aplicaciones web
  • Empresa e iniciativa emprendedora
  • Módulo de FCT (Formación en Centros de Trabajo)


Agrupamientos y organización espacial:

  • Gran grupo o grupo-clase (para revisión de avances)
  • Grupos fijos (3 o 4 personas)
  • Equipos flexibles (3 o 4 personas)
  • Trabajo individual (tareas específicas)

Temporalización: 4 semanas, es decir, 8 sesiones de 2 horas.

Descripción de las tareas/actividades. Recursos y herramientas tecnológicas

TAREA 1: Presentación del reto y análisis de requisitos del banco de alimentos

Temporalización: 1 sesión (2 horas)

Descripción:
Se presenta al alumnado el contexto del banco de alimentos y el reto: desarrollar una aplicación web realista que gestione usuarios, productos, donaciones y stock. Se parte de un dossier que simula o incluye necesidades reales.

Actividades del alumnado:

  • Analizar el dossier del banco de alimentos mediante una infografía: ¿qué necesita, quién lo usará, qué funcionalidades debe tener?
  • Formular dudas, proponer ideas, discutir funcionalidades mínimas viables.
  • Crear un documento resumen del análisis de requisitos.
  • Proponer mejoras o adaptaciones al contexto real.

Recursos:

  • Dossier de requisitos funcionales (PDF o editable)
  • Proyector o pizarra digital para la presentación
  • Google Docs para lluvia de ideas grupal
  • Plataforma del aula Teams para compartir documentos

TAREA 2: Diseño de la base de datos y arquitectura MVC del sistema

Temporalización: 2 sesiones (4 horas)

Descripción:
Se diseña el modelo de base de datos con un diagrama ER y se traza la arquitectura general de la aplicación (Modelo-Vista-Controlador), conectando entidades y rutas del backend.

Actividades del alumnado:

  • Definir las tablas necesarias: usuarios, productos, donaciones, stock, etc.
  • Crear el diagrama entidad-relación y verificar claves primarias/foráneas.
  • Planificar la estructura de carpetas del proyecto y cómo se organizarán los controladores, modelos y vistas.
  • Validar el diseño con el docente antes de comenzar la implementación.

Recursos:

  • Software MySQL
  • Herramientas de diagramación (Draw.io)
  • Guía de arquitectura MVC adaptada al lenguaje/stack usado
  • Plantilla de planificación del backend

TAREA 3: Implementación por módulos: usuarios, productos, donaciones, stock

Temporalización: 2 sesiones (4 horas)

Descripción:
Los estudiantes desarrollan los distintos módulos del sistema: gestión de usuarios, productos disponibles, donaciones registradas y stock actualizado. Se puede usar Spring Boot, Node.js con Express u otro stack similar.

Actividades del alumnado:

  • Crear endpoints REST para cada entidad.
  • Implementar operaciones CRUD con validaciones básicas.
  • Utilizar ORM (JPA/Hibernate o Sequelize/Mongoose) para interactuar con la base de datos.
  • Probar los endpoints con Postman, realizar un videotutorial de esta herramienta y dejar comentarios en el código.
  • Realizar commits regulares en un repositorio de GitHub.

Recursos:

  • IDE: IntelliJ, VS Code
  • Servidor local: Tomcat, XAMPP, Node.js
  • Spring Boot, Node.js + Express
  • Control de versiones (Git/GitHub)
  • Postman

TAREA 4: Revisión de seguridad, testeo y documentación

Temporalización: 2 sesiones (4 horas)

Descripción:
Los estudiantes aseguran la aplicación añadiendo autenticación (JWT o sesiones), testean funcionalidades clave y documentan los endpoints con Swagger/OpenAPI o similar.

Actividades del alumnado:

  • Proteger las rutas sensibles con middleware de autenticación.
  • Crear casos de prueba unitarios o pruebas funcionales básicas.
  • Documentar todos los endpoints, parámetros, respuestas y errores comunes.
  • Revisar el código en equipo para asegurar la legibilidad y modularidad.

Recursos:

  • Librerías de seguridad (Spring Security, JWT)
  • Herramientas de test (JUnit, Postman)
  • Swagger UI / OpenAPI
  • Documentación en Markdown o Word

TAREA 5: Presentación y despliegue del proyecto en servidor local/remoto

Temporalización: 1 sesiones (2 horas)

Descripción:
El alumnado presenta su aplicación en funcionamiento, despliega el proyecto en un servidor local (Tomcat, Apache) o en la nube (Heroku), y explica sus decisiones de desarrollo.

Actividades del alumnado:

  • Subir el código final a GitHub con README completo.
  • Configurar el entorno de despliegue (puertos, entorno, variables).
  • Realizar una presentación oral (máx. 10 min) mostrando el flujo de uso de la app.
  • Recibir feedback del docente y compañeros/as.

Recursos:

  • Servidor de despliegue (Heroku)
  • Docker
  • Diapositivas para la presentación (Canva)
  • Proyector para la demo
